# Compliance

The compliance domain maps the organization's controls to the **frameworks and obligations** it must satisfy, and provides a single crosswalk so that one control can demonstrate compliance across multiple frameworks.

## Folder Map

| Path | Description |
| --- | --- |
| frameworks/iso-27001/ | ISO/IEC 27001 Annex A Statement of Applicability |
| frameworks/soc2/ | SOC 2 Trust Services Criteria mapping |
| frameworks/nist-csf/ | NIST Cybersecurity Framework profile |
| control-mappings/ | Cross-framework control crosswalk (control-crosswalk.csv) |

## How to Use

1. Select the framework(s) that apply to your obligations.
2. Complete the applicability/profile document, marking each control as implemented, partial, or not applicable.
3. Use control-mappings/control-crosswalk.csv to see how a single control satisfies multiple frameworks.
4. Link each control to its evidence in audit/evidence/evidence-tracker.csv.

## Supported Frameworks

| Framework | Focus |
| --- | --- |
| ISO/IEC 27001 | Information Security Management System (ISMS) certification |
| SOC 2 | Trust Services Criteria for service organizations |
| NIST CSF | Risk-based cybersecurity outcomes (Identify, Protect, Detect, Respond, Recover, Govern) |

## Status Values

Use Implemented, Partial, Planned, or Not Applicable for each control, with a short justification.
