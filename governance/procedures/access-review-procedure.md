# Access Review Procedure

| Field | Value |
| --- | --- |
| Document ID | GOV-PROC-001 |
| Owner | Security Team |
| Status | Approved |
| Version | 1.0 |
| Frequency | Quarterly (privileged: monthly) |
| Supports | Access Control Policy (GOV-POL-003) |

## 1. Purpose

To verify that access rights remain appropriate and to remove access that is no longer required.

## 2. Scope

All in-scope systems holding Confidential or Restricted data, and all privileged access.

## 3. Procedure

1. Generate the access report for each in-scope system (users, roles, and entitlements).
2. Distribute reports to the relevant resource or line-manager reviewers.
3. Reviewers confirm, modify, or revoke each entitlement and record a decision.
4. The security team actions revocations and modifications within the agreed SLA.
5. Capture evidence (report, reviewer sign-off, ticket numbers) in the evidence tracker.
6. Escalate overdue reviews or unresolved findings to the CISO.

## 4. Inputs and Outputs

Inputs: system access reports, HR joiner/mover/leaver data. Outputs: completed review records, change tickets, and audit evidence.

## 5. Records

Review records are retained for at least the audit retention period and stored in audit/evidence/.

## 6. Roles

| Role | Responsibility |
| --- | --- |
| Security Team | Coordinate the review and action changes |
| Resource/Line Managers | Review and approve access |
| CISO | Oversight and escalation |
