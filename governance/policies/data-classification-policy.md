# Data Classification Policy

| Field | Value |
| --- | --- |
| Document ID | GOV-POL-004 |
| Owner | CISO |
| Approver | Executive Leadership |
| Status | Approved |
| Version | 1.0 |

## 1. Purpose

To ensure information is classified and handled according to its sensitivity and business value.

## 2. Scope

Applies to all information created, received, stored, or transmitted by the organization in any form.

## 3. Classification Levels

| Level | Description | Examples |
| --- | --- | --- |
| Public | No harm if disclosed | Marketing material, public website |
| Internal | Limited harm if disclosed | Internal memos, org charts |
| Confidential | Significant harm if disclosed | Contracts, source code, customer data |
| Restricted | Severe harm if disclosed | Secrets, regulated personal data, credentials |

## 4. Handling Requirements

| Level | Storage | Sharing | Disposal |
| --- | --- | --- | --- |
| Public | No restriction | No restriction | Standard |
| Internal | Access-controlled | Internal only | Standard delete |
| Confidential | Encrypted at rest | Need-to-know, approved channels | Secure deletion |
| Restricted | Encrypted, restricted access | Explicit owner approval | Verified destruction |

## 5. Roles

Data owners assign classification. Data custodians implement controls. All users handle data per its classification.

## 6. Labeling

Where feasible, documents and systems should be labeled with their classification.

## 7. Review

Reviewed at least annually.
