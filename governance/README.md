# Governance

The governance domain defines the **direction, authority, and rules** of the information-security program. It contains the mandate (charters), the approved policies, the technical standards that make policies enforceable, and the procedures that operationalize them.

## Folder Map

| Path | Description |
| --- | --- |
| governance/charters/ | Program and committee charters that establish authority and scope |
| governance/policies/ | Approved, board/management-level policies (the "what" and "why") |
| governance/standards/ | Mandatory technical standards (the "how much / how strong") |
| governance/procedures/ | Step-by-step operational procedures (the "how") |

## Files

| Path | Purpose |
| --- | --- |
| charters/security-program-charter.md | Establishes the security program, its mission, authority, and roles |
| policies/information-security-policy.md | The overarching information-security policy |
| policies/acceptable-use-policy.md | Rules for acceptable use of organizational assets |
| policies/access-control-policy.md | Principles and rules for granting and revoking access |
| policies/data-classification-policy.md | Classification levels and handling requirements |
| standards/password-standard.md | Minimum password and authentication requirements |
| procedures/access-review-procedure.md | Periodic user access review procedure |

## Policy Hierarchy

Charter -> Policy -> Standard -> Procedure. Charters grant authority; policies state intent; standards make intent measurable; procedures make standards repeatable.

## Ownership and Review

Each document names an Owner and an Approver and is reviewed at least annually or after any significant change to the business, technology, or threat landscape.
